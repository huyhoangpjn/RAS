# Ray Augmented Supervision (RAS) for 3D Object Detection

## Overview

**Ray Augmented Supervision (RAS)** is a plug-and-play one-to-many (O2M) training framework that directly supervises operated queries along camera rays. It consists of three complementary components:

1. **Ray-based Ground-truth Generation and Assignment (RGG + PSA):** For each ground-truth object, pseudo-labels are generated by uniformly sampling points along the camera ray at varying depths. A quality-based greedy matching assigns hard FPs (near-ray predictions) as additional positives, tolerating large depth errors that occur in early training.

2. **Proximity-aware Box Weighting (PBW):** Among the multiple candidates assigned to a single ground truth, those with smaller translation error receive stronger supervision. This prevents distant, high-loss samples from dominating the regression signal.

3. **O2M Adapted Architecture:** The attention order is changed to `temporal_attn → cross_attn → self_attn → FFN`. Cross-attention paired with the O2M classification head proposes candidates, while self-attention guided by the O2O head suppresses duplicates. NMS at inference provides further FP suppression that self-attention alone cannot achieve in 3D.

RAS is applied to a deformable-attention variant of [StreamPETR](https://github.com/exiawsh/StreamPETR) (termed **RepDETR3D**) and achieves the following results on nuScenes validation:

| Method | Backbone | mAP | NDS | Add. Q |
|:---:|:---:|:---:|:---:|:---:|
| Baseline (StreamPETR reprod.) | R50 | 44.7 | 54.7 | 0 |
| GroupDETR† | R50 | 45.6 | 55.2 | 1800 |
| H-DETR† | R50 | 45.5 | 55.2 | 1800 |
| **RAS (ours)** | **R50** | **46.3** | **55.2** | **0** |
| Baseline (StreamPETR reprod.) | VoV-99 | 49.2 | 58.0 | 0 |
| **RAS (ours)** | **VoV-99** | **50.3** | **58.7** | **0** |

*† O2M applied to StreamPETR using auxiliary training-only queries.*

RAS is also complementary to [RayDN](https://github.com/LiewFeng/RayDN) query denoising:

| Method | w/ RAS | mAP | NDS |
|:---:|:---:|:---:|:---:|
| RAS | — | 46.3 | 55.2 |
| RayDN | ✗ | 46.9 | 56.3 |
| RayDN | ✓ | **47.1** | **56.5** |
| RayDN + depth | ✗ | 47.5 | 56.6 |
| RayDN + depth | ✓ | **47.9** | 56.3 |

## Installation and Environment Setup

Follow the **StreamPETR** setup guide:

- [Environment Setup](https://github.com/exiawsh/StreamPETR/blob/main/docs/setup.md)

Key dependencies:

```
Python >= 3.8
CUDA == 11.2
PyTorch == 1.9.0
mmdet3d == 1.0.0rc6
flash-attn == 0.2.2 (optional)
```

Step-by-step:

```bash
conda create -n streampetr python=3.8 -y
conda activate streampetr

# Install PyTorch
pip install torch==1.9.0+cu111 torchvision==0.10.0+cu111 torchaudio==0.9.0 \
    -f https://download.pytorch.org/whl/torch_stable.html

# Install flash-attn (optional, skip if not supported by your GPU)
pip install flash-attn==0.2.2

# Install mmdet3d dependencies
pip install mmcv-full==1.6.0 -f https://download.openmmlab.com/mmcv/dist/cu111/torch1.9.0/index.html
pip install mmdet==2.28.2
pip install mmsegmentation==0.30.0

# Install mmdetection3d
cd mmdetection3d
git checkout v1.0.0rc6
pip install -e .
```

## Training and Inference

All scripts are run from the repository root. Use `tools/dist_train.sh` for multi-GPU training and `tools/dist_test.sh` for evaluation.

### Table 1 — Comparison with Existing O2M Methods

These configs reproduce the results in **Table 1** of the paper.

#### GroupDETR baseline (R50, 60ep) — *repdetr3d_r50_group_detr*

```bash
# Training (2 GPUs)
bash tools/dist_train.sh \
    projects/configs/RepDETR3D/repdetr3d_r50_group_detr.py 2

# Evaluation
bash tools/dist_test.sh \
```

### Optimal evaluation results

RAS introduces NMS at inference time. Because the standard in-training evaluation uses a fixed NMS threshold, the reported training-time mAP is **sub-optimal**. We separately search for the best class-specific NMS thresholds on the validation set and provide those results under `logs/.../optimal_res/`.

Optimal class-specific NMS thresholds:

| Class index | Class name | NMS threshold |
|:---:|:---:|:---:|
| 0 | car | 0.5 |
| 1 | truck | 0.4 |
| 2 | construction_vehicle | 0.2 |
| 3 | bus | 0.6 |
| 4 | trailer | 0.2 |
| 5 | barrier | 0.5 |
| 6 | motorcycle | 0.5 |
| 7 | bicycle | 0.5 |
| 8 | pedestrian | 0.8 |
| 9 | traffic_cone | 0.5 |

To reproduce the optimal results, set `cls_spec_nms` in the `NMSCoder` bbox coder of the config accordingly (see the commented line in `repdetr3d_r50_ras.py`):

```python
bbox_coder=dict(
    type='NMSCoder',
    ...
    cls_spec_nms={0: 0.5, 1: 0.4, 2: 0.2, 3: 0.6, 4: 0.2,
                  5: 0.5, 6: 0.5, 7: 0.5, 8: 0.8, 9: 0.5},
    ...
)
```
